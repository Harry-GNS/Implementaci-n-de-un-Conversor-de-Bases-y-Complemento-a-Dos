# Conversor de Bases y Complemento a Dos

Repositorio para la actividad: Conversor de bases y complemento a dos.

Estructura creada:

- `src/` - código fuente del conversor
  - `conversor.py` - archivo principal (esqueleto)
- `tests/` - pruebas unitarias
  - `test_conversor.py` - pruebas básicas (esqueleto)
- `requirements.txt` - dependencias (vacío por ahora)
- `.gitignore` - archivos a ignorar por git

Próximos pasos:

1. Implementar las funciones en `src/conversor.py` siguiendo las fases de la actividad.
2. Añadir pruebas extra en `tests/` y ejecutar con `pytest`.
3. Inicializar el repositorio git con `git init` y subir a un remoto si se desea.

Notas:

- El código principal está pensado para permanecer en un solo archivo `conversor.py` según la preferencia indicada.
- El contenido del archivo está en español (sin comentarios en el código) y esqueleto para implementar.
